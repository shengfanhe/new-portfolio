Marketing Cloud Journey Builder

Deliver cross-channel personalized experiences at every step of the customer lifecycle with campaign management.

About the product

All-in-one marketing automation driven by all kinds of customer data.


Salesforce conceptualizes the progression of interactions as a journey through customer relationship stages including reach, acquisition, conversion, retention and loyalty. Journey Builder is designed to allow marketers to customize interactions through those stages based on customer needs, desires and preferences, their demographics and real-time input from their behavior.

My role

Owner of all data features' design


Data is the fuel of digital marketing, every campaign starts with customer data. My role in this product team is to be responsible for the UX/UI design of all data related features, inluding data setup and import, audience creation, API event triggers and content personalization. I worked closely with PMs to define product functionalities, and pushed the ideas towards solid solutions by a lot of prototyping.

Timeline

The project started from May 2017, when the company decided to re-think Marketing Cloud. Through the POC, intensive concpetual designs, we clarified the direction that has journey in the center. We designed the framework to align functions with Journey Builder. The detail design and iterations last to the end of 2018.

Marketing Cloud Journey Builder Demo

Audience: a unified starting point of configuring a journey.

Marketing Cloud used to have many different data objects, configuring the data entry is a tech-heavy task when building a journey. Now we're building a unified data entry as audience, to bring a consistent, user friendly experience with more advanced fucntions built in.

Building an audience on the fly.

With the idea "every marketing campaign is a journey", we try to build Audience configuration inside of the journey creation flow. To embed in such a big function which used to be a standalone product, I created the "standard configuration modal" UI framework to accommodate not only audience but also many other function use cases.

Expression Builder: create SQL query in the UI.

One of the key components of an audience is the the rules defined by the filters, based on the customer data. Marketers relied heavily on the IT support in the past to write the SQL query to define the audience, now we try to make up UI functionality so that marketers can do this themselves.

A special type of journey: real-time triggered send.

Based on the journey framework, I designed the triggered send configuration flow to keep the consistency with the regular journey. Real-time send has very different configuration steps, so it has a step-by-step guide. A triggered send can be translated to a regular journey.

Setup Assitant: an onboarding experience for marketers.

As a product aiming to provide the best self-service experience, we have to make sure user can do all of those fundation works that get the product up and running. So we created a Setup Assistant to help user setting up the product and data, while learning how to use the product along the way.
